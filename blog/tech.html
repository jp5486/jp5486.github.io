Regular Expressions

Nov 1, 2015

Regular expressions are a very powerful tool that can be used across languages to help search out patterns in your code.
Although the subject can only be mastered with practice, here is a short summary of how to reference different search patterns with a cheat sheet image at the bottom. Enjoy

So what is the purpose of regular expressions? They are use to search for literal orders of string characters and digits.
abc or 123 will look for instances of the 3 letters or numbers in that specific order.

\d searches for any digit
\D any Non-digit character like punctuation

\w searches for any alphanumeric character
\W any Non-alphanumberic character

The . character is a general joker/wildcard

Surrounding letters with brackets will look for those sepecfic letters
[fgh] would look for any of the letters: 'f', 'g', 'h' in the pattern.

Using a hat ^ INSIDE the brackets will exclude letters from a search
[^ijk] would skip words with the letters 'i' 'j' or 'k' from your search.

Ranges can also be set within brackets:
[4-9] will search a character for the digits 4 through 9
[^a-g] will search for all letters except a through g

Note that letters are case sensitive
[A-C] will only get the words containing Upper Case letters 'A','B','C'

Searching for multiple occurances of a letter is fairly simple
\x\x\x or x{3} will search for 3 instances of the lower case letter 'x' connected
Also you can set a range inside the curly braces. x{3,6} will look for between 3 and 6 occurances of the letter x
But there are other ways as well:

A * character represents 0 or more of a charater while
The + character represents 1 or more and makes more sense to use.
B+ looks for 1 or more Upper Case letter B

The ? character is a little trying and is used to signal an optional character search. It is similar to using *. The results will show with or without the preceding character to the ?
smiles? - the s is optional and the search will return smile and smiles

When needing to reference whitespace use "\s".

Using a hat ^ without brackets at the beginning and a dollar sign $ at the end will search for an exact match for the contents. I would compare this to using quotes on a search engine.
^Clear Example$ would literally search for the exact string inside the special characters in that casing instead of returning partial matches.

Parentheses can be used to check for subpatterns such as file names.
^(screenshot.*)\.png$ searches for files starting with the wordscreenshot but may have a longer name and are also appended with a .png file extension.
The contents in the parentheses can then be used for futher data processing and You can also nest parentheses within each other for extra data processing.

Lastly we can add a pipe | inside of a parentheses to signify the or conditional statement which can add to your search results.

For some practical examples I reccomend regexone.
