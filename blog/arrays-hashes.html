Arrays and Hashes

Oct 4, 2015

Today we will discuess the differences between arrays and hashes and provide some examples of how to use each

By definition an array is an ordered, integer-indexed collection of objects; and a hash is a dictionary-like collection of uniqure keys and their values.
Now what does that mean?

In its simpliest sense, an array is a store set of objects. These objects can be strings, integers, floats or even other arrays. A hash is a more defined version of an array in that is has a key and value pair, similar to a dictionary. In a dictionary you have a word and a description of that word. The key is like a word and the value being the description. You access the value(or definition) of a key(or word).

Lets look at some examples of each.

Here is an example of an array called "my_family". I have stored 5 strings inside. At the 0 placeholder(the very first place in the list since we start counting from 0) I have stored the string "my name". "father" is in the 1 placeholder

Now lets look over some Hash Examples

Here we have a hash called my_family_hash with various assignments for each name. The definition for my_name is Jeremy, father is Ernie, mother is Carol, etc. Again, we call the entire hash variable on line 11 and the definition for the first assignment on line 13.

Here are the responses from our terminal. We see the entire hash printed out and the first definition on the next line, Perfect!

Even though this is not a comprehensive article, we can start see the subtle differences between an array and hash. You can see the key:value or word:definition relationship in an array as well, the only difference is you cannot chose the keys. They are fixed as integers counting upwards from 0. In a hash you are able to write the key and value for each one.

Please refer to the Ruby documentation online for a more in depth breakdown of all the features that arrays and hashes are capable of.
